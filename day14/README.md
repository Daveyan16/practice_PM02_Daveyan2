# Проект: Сервисный слой банковского приложения (Вариант 6)

## 1. Описание проекта

Проект представляет собой реализацию сервисного слоя (*Service Layer*) для предметной области «Банковские счета и переводы».

**Цель:** Изучить принципы построения сервисного слоя, освоить паттерны *Repository*, *Service*, *Dependency Injection* и научиться покрывать бизнес-логику юнит-тестами.

### Реализованные функции:
*   Создание банковских счетов.
*   Пополнение счета (*Deposit*).
*   Снятие средств (*Withdrawal*).
*   Переводы между счетами с проверкой баланса и лимитов.
*   Формирование выписки по операциям.

## 2. Архитектура и решенные проблемы

В рамках задания были решены архитектурные проблемы стартового кода:
1.  **Нарушение принципа единственной ответственности (*SRP*):** Бизнес-логика вынесена из репозиториев в отдельный сервисный слой.
2.  **Жесткая связанность (*Coupling*):** Внедрение зависимостей (*DI*) реализовано через конструктор сервиса.
3.  **Отсутствие транзакционности:** Обеспечена атомарность операций (в памяти — через сохранение старого состояния) и обработка ошибок с откатом изменений.
4.  **Недостаточное покрытие тестами:** Написаны юнит-тесты для основных сценариев.

## 3. Структура проекта

Код разделен на логические слои согласно принципам чистой архитектуры:
*   **`banking/entities.py`**: Модели данных (сущности `Account`, `Transaction`).
*   **`banking/repositories.py`**: Абстрактные интерфейсы репозиториев (*Protocols*).
*   **`banking/in_memory_repositories.py`**: Конкретные реализации репозиториев для работы в оперативной памяти.
*   **`banking/services.py`**: Сервисный слой, содержащий всю бизнес-логику.
*   **`banking/exceptions.py`**: Собственная иерархия доменных исключений.
*   **`tests/test_banking_service.py`**: Юнит-тесты для сервисного слоя.
*   **`main.py`**: Пример использования сервисов.

## 4. Запуск и тестирование

Для запуска проекта выполните следующие команды в корневой папке проекта:

1.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```
2.  **Запустите тесты с покрытием кода:**
    ```bash
    pytest -v --cov
    ```
    Эта команда запустит все тесты и покажет отчет о покрытии.

3.  **Запустите демонстрационный пример:**
    ```bash
    python main.py
    ```
    Эта команда выведет в консоль результат выполнения операций.
