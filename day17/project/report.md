# Практическая работа №17

## Тема работы

Комплексное тестирование и валидация программного обеспечения с использованием готовых тестовых наборов.

## Вариант №6

### Цель работы

Освоить методы тестирования программного обеспечения с использованием библиотеки pytest, научиться выявлять дефекты в коде, создавать unit-тесты и анализировать результаты тестирования.

---

# Ход выполнения работы

В ходе выполнения практической работы был рассмотрен модуль `notification.py`, предназначенный для отправки уведомлений пользователям.

Была создана структура проекта, включающая исходный код программы, тесты и отчетную документацию. Для работы были установлены необходимые зависимости: `pytest`, `pytest-cov`, `mutmut` и `pytest-xdist`.

После анализа варианта №6 были выявлены основные проблемы модуля:

* отсутствие обработки ошибок при отправке Email;
* отсутствие обработки таймаутов при отправке SMS;
* отсутствие проверки результата отправки Push-уведомлений.

Для устранения недостатков был разработан исправленный вариант модуля `notification.py`.

Далее был создан файл `test_notification.py`, содержащий набор unit-тестов с использованием объектов `Mock`. Тесты предназначены для проверки как успешных сценариев работы функций, так и обработки ошибок внешних сервисов.

Во время запуска тестирования были обнаружены проблемы со структурой проекта и импортом модулей. Ошибки были устранены путем настройки путей импорта и корректировки структуры каталогов проекта.
